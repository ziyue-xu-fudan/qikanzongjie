# 构建 Streamlit 文献筛选交互应用

为了提供比 Excel 更高效、舒适的筛选体验，我们将开发一个名为 `app_screening.py` 的 Web 应用。

## 1. 核心功能设计

*   **数据源**: 读取 `Literature_Screening_List.xlsx`。
*   **界面布局**:
    *   **Sidebar**: 进度条（已筛/总数）、过滤器（只看未筛/已筛）、导出按钮。
    *   **Main Area**:
        *   文章元数据卡片（期刊、时间、DOI）。
        *   **大字号标题**。
        *   **高亮摘要**：自动高亮 "prospective", "cohort", "registry" 等关键词，帮助快速抓取重点。
    *   **操作区**: 两个大按钮 "✅ Include" 和 "❌ Exclude"，支持键盘快捷键（如左/右方向键）。
*   **状态管理**: 使用 `st.session_state` 记录当前浏览的文章索引，点击按钮后自动跳转下一篇。
*   **持久化**: 每次操作立即回写 Excel 文件，防止数据丢失。

## 2. 技术栈
*   Streamlit (UI 框架)
*   Pandas (数据处理)
*   OpenPyXL (Excel 读写)

## 3. 实施步骤
1.  创建 `app_screening.py`。
2.  实现数据加载与保存逻辑。
3.  构建 UI 界面与高亮逻辑。
4.  添加键盘快捷键支持（提升效率的关键）。
5.  启动服务供您使用。

## 4. 预期交付
一个本地运行的网页链接，您点开即可开始像“刷 Tinder”一样高效筛选文献。
